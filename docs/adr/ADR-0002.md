# ADR-0002: Explicit Lifecycle Transition Matrix

## Status
Approved

## Date
2026-06-25

## Context
Artifacts go through multiple stages (Draft -> Processing -> Metadata -> Published).

## Problem
Ordinal comparisons (e.g., `lifecycle > PROCESSING`) are fragile because they assume a linear flow that might change. Implicit transitions make it hard to reason about valid states.

## Decision
Implement an explicit transition matrix in `ArtifactLifecycle`. Every transition must be checked via `canTransitionTo(next)`.

## Alternatives Considered
- **Ordinal Enums**: Rejected because the flow is not strictly linear (e.g., recovery paths).
- **State Machine Library**: Rejected to keep the core model simple and dependency-free.

## Consequences
- Impossible to accidentally regress state without explicitly allowing it.
- Clear documentation of the allowed lifecycle paths.
