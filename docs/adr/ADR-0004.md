# ADR-0004: Atomic Recording Finalization

## Status
Approved

## Date
2026-06-25

## Context
When a recording finishes, we must update `durationMs`, `durableBytes`, `lifecycle`, and `updatedAt`.

## Problem
Partial updates (e.g., updating lifecycle but failing to save duration) leave the draft in a corrupt state that workers might fail to process correctly.

## Decision
Perform all finalization updates within a single Room database transaction (`draftsDatabase.withTransaction`).

## Alternatives Considered
- **Multiple sequential updates**: Rejected because it's not crash-safe.

## Consequences
- Guaranteed atomicity: either all metadata is saved and the state transitions, or nothing happens.
- Prevents "half-finalized" drafts.
