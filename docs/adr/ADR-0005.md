# ADR-0005: Publishing Flow Architecture Invariants

## Status
Approved

## Date
2026-06-25

## Context
The publishing flow is a critical path involving recording, processing, review, and upload.

## Problem
As the app evolves, "shortcuts" might be taken that violate core assumptions (e.g., workers running on unfinalized drafts), leading to hard-to-debug race conditions.

## Decision
Establish a permanent set of architectural invariants (rules that must never be broken).

## Alternatives Considered
- **Strict Code Reviews only**: Rejected because invariants provide a clear checklist that can be verified by both humans and AI agents.

## Consequences
- Clear guardrails for future development.
- Reduced architectural drift.
