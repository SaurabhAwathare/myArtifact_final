# ADR-0003: Single Source of Truth for Draft State

## Status
Approved

## Date
2026-06-25

## Context
Draft data (title, audio path, lifecycle) is used across multiple screens and background workers.

## Problem
Keeping state in memory (ViewModels) or passing it via navigation arguments leads to stale data and "split-brain" scenarios where different parts of the app see different versions of a draft.

## Decision
The Room database (`DraftDao`) is the only authoritative source of truth for draft state. UI components must observe the database (via Flow) rather than holding permanent copies of the data.

## Alternatives Considered
- **Memory-only cache**: Rejected because it doesn't survive process death and makes recovery impossible.

## Consequences
- UI is always in sync with the persisted state.
- Workers and UI operate on the same data.
- Slightly higher latency for updates (mitigated by UI buffers, see ADR-0006).
