# ADR-0006: Local Title Buffer with Debounced Persistence

## Status
Approved

## Date
2026-06-25

## Context
Writing to Room on every keystroke in a title text field is inefficient and can cause UI stuttering.

## Problem
A direct UI-to-Room binding feels laggy. However, holding state *only* in the ViewModel violates the SSOT principle (ADR-0003).

## Decision
Use a temporary local buffer (`MutableStateFlow`) in the ViewModel for immediate UI feedback, combined with a debounced (500ms) write to the `RecordingRepository`. The buffer is cleared only after successful persistence.

## Alternatives Considered
- **Direct binding**: Rejected due to performance concerns.
- **Manual "Save" button**: Rejected to maintain the "ambient/automatic" user experience.

## Consequences
- Zero-latency UI response.
- Eventual consistency with SSOT.
- Crash-safe (at most 500ms of typing is lost).
